# QA 라이브 과제 — 장바구니 수량 변경 기능

> **⏱ 제한 시간 40분** · 현장에서 시간 재고 푸는 라이브 시험 형식
> 자동화 코드 불필요 · 형식 자유

---

## ⚠️ 라이브 시험 공략법 (먼저 읽기)

- **다 푸는 게 목표가 아니다.** 시간 안에 핵심을 빠르게 잡고, 우선순위 판단력을 보여주는 게 평가 포인트다.
- 막히면 붙잡지 말고 넘어간 뒤, 마지막에 **"시간이 더 있으면 추가하겠다"** 항목으로 적어라. (의도적으로 인지하고 넘어간 것 = 가점)
- 표 정렬·문장 다듬기에 시간 쓰지 말 것. 내용 먼저, 모양 나중.
- 권장 시간 배분: 요구사항 분석 8분 / TC 설계 22분 / 리스크 7분 / 검토·정리 3분

## 배경

당신은 이커머스 플랫폼 **NS몰**의 QA 엔지니어다.

**장바구니에서 상품 수량을 변경하는 기능**을 검증해야 한다. 아래는 기획 명세다.

---

## 기능 명세 (v1.0)

### 1. 수량 변경
- 장바구니의 각 상품은 **수량 조절 UI**(`-` 버튼 / 숫자 입력 / `+` 버튼)를 가진다.
- 사용자는 버튼 또는 **직접 입력**으로 수량을 변경할 수 있다.
- 수량 변경 시 **해당 상품 금액**과 **장바구니 총 금액**이 즉시 갱신된다.

### 2. 수량 제약
- 최소 수량은 **1개**다.
- 상품의 **재고 수량**을 초과해 담을 수 없다.
- 일부 상품은 **1인 최대 구매 수량** 제한을 가진다. (예: 1인 최대 5개)

### 3. 기타
- 수량 변경과 별개로, 상품을 장바구니에서 **삭제**하는 버튼이 따로 있다.
- **품절** 상품은 장바구니에서 수량을 늘릴 수 없다.

---

## 제출물 (라이브 — 가볍게)

### 1. 요구사항 분석 (간략)
명세에서 **모호하거나 누락된 부분 3개 이상**을 빠르게 나열하고, 테스트를 위한 **가정**을 1~2줄로 적어라.

### 2. 테스트 케이스 (핵심 위주)
이 기능의 테스트 케이스를 작성하라.
- **정상 / 경계 / 예외**를 구분
- 각 케이스에 **우선순위(H/M/L)**
- 컬럼: `구분` | `테스트 명` | `입력/조건` | `기대결과` | `우선순위`
- *시간이 부족하면 우선순위 H부터 채우고, 나머지는 제목만이라도 나열할 것*

### 3. 리스크 (Top 1~2)
가장 결함 위험이 크거나 임팩트가 큰 영역 **1~2개**와 이유.

### 4. (시간 남으면) 더 할 것
시간 제약으로 다루지 못한 항목을 목록으로 남겨라.

---

> 끝나면 작성한 만큼 그대로 가져와. 라이브 기준(시간 내 우선순위 판단)으로 리뷰해줄게.

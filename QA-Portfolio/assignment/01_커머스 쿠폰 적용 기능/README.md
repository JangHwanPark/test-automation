## 기능 명세 (v1.0)
### 1. 쿠폰 종류
- **정액 할인 쿠폰**: 고정 금액을 할인한다. (예: 5,000원 할인)
- **정률 할인 쿠폰**: 주문 금액의 일정 비율을 할인한다. (예: 10% 할인)
- 정률 할인 쿠폰은 **최대 할인 한도**를 가질 수 있다. (예: 10% 할인, 최대 20,000원)
- 각 쿠폰은 **유효기간**(시작일 ~ 종료일)을 가진다.

### 2. 적용 규칙
- 쿠폰을 적용하려면 **최소 주문 금액** 조건을 충족해야 한다. (예: 30,000원 이상 구매 시 사용 가능)
- 할인은 **상품 금액 합계**에 대해 적용된다.
- 사용자는 결제 단계에서 보유 쿠폰 목록 중 **하나를 선택**해 적용할 수 있다.
- 적용한 쿠폰은 다시 **해제**할 수 있다.

### 3. 화면 동작
- 쿠폰을 적용하면 최종 결제 금액이 **즉시** 갱신된다.
- 적용 불가능한 쿠폰은 목록에서 **비활성화 상태**로 표시된다.

## 제출물
다음 **4가지**를 작성해 제출하세요.

### 1. 요구사항 분석
위 명세에서 **모호하거나 누락되었다고 판단한 부분**을 정리하고,
테스트를 진행하기 위해 당신이 세운 **가정(assumption)**을 명시하세요.

### 2. 테스트 케이스 설계
이 기능에 대한 테스트 케이스를 **표 형식**으로 작성하세요.
- **정상 / 경계값 / 예외** 케이스를 구분할 것
- 각 케이스에 **우선순위(High / Medium / Low)** 부여
- 필수 컬럼: `TC ID` | `구분` | `사전조건` | `테스트 절차` | `입력값` | `기대결과` | `우선순위`

### 3. 리스크 분석
이 기능에서 **결함 발생 가능성이 높거나, 결함 발생 시 비즈니스 임팩트가 큰 영역**을
3가지 선정하고, 선정 이유를 설명하세요.

### 4. 스펙 리뷰
명세서 자체에 존재하는 **결함·위험(논리적 허점, 정의되지 않은 동작)**을
발견한 대로 개발/기획팀에 전달할 형태로 보고하세요.

# 과제 P-01 · 한정판 입찰 기능 출시 전 시나리오 우선순위

| 항목 | 내용 |
|---|---|
| 카테고리 | P · 시나리오 우선순위 |
| 난이도 | 중급 |
| 권장 소요 시간 | 35분 |
| 대상 회사 | 크림 (KREAM) |
| 학습 목표 | 신규 기능 출시 전 회귀 영역까지 포함한 우선순위 산정 |
| 평가 포인트 | P0 정의 엄격성 · 회귀 영역 포함 · 비기능(성능/보안) 시나리오 포함 |

## 기능 요약 (출시 대상)
한정판 상품 **구매 입찰** 기능 신규 출시. 기존에는 즉시 구매만 가능했음.

- 사용자는 사이즈별 희망 입찰가와 입찰 유지 기간을 입력해 구매 입찰을 등록할 수 있다.
- 동일 사이즈의 판매 입찰가와 조건이 일치하면 거래가 자동 체결된다.
- 체결 후 구매자는 결제를 진행해야 한다.
- 결제 완료 시 판매자 발송 대기 → 검수 → 배송 흐름을 탄다.
- 입찰 유지 기간이 만료되면 입찰은 자동 취소된다.
- 출시 영역: App / Web 동시

## 작성 문제
위 기능 출시 전 진행할 테스트 시나리오를 우선순위와 함께 작성하시오.

## 작성 조건
- **최소 15개 이상** 시나리오
- 신규 기능(입찰)뿐만 아니라 **회귀 영역(즉시 구매 / 결제 / 알림 등)** 포함
- 비기능 시나리오(성능, 보안, 동시성) **최소 2개 이상** 포함
- 우선순위 분포: P0 20% 내외, P1 30~40%, P2 30%, P3 10~20% 목표
- 각 P0 시나리오는 "왜 출시 차단 사유인지" 근거 명시

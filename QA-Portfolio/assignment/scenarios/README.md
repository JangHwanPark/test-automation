# P · Test Scenarios / Prioritization · 시나리오 우선순위

주어진 신규 기능 / 변경 사항에 대해 **출시 전 어떤 시나리오를 어떤 우선순위로** 검증할지 산정하는 카테고리.

## 표준 양식

| ID | 시나리오 | 검증 목적 | 우선순위 | 분류 | 예상 소요 | 근거 / 리스크 |
|---|---|---|---|---|---|---|
| SC-01 | 정상 송금 (1만원) | 핵심 골든 패스 | P0 | Smoke | 5분 | 핵심 사용자 흐름. 깨지면 서비스 중단 |
| SC-02 | 1,000만원 초과 시 영상 인증 진입 | 정책 조건 | P0 | Regression | 15분 | 보안/규제 이슈 |

### 컬럼 정의
| 컬럼 | 설명 |
|---|---|
| ID | `SC-NN` 순번 |
| 시나리오 | 한 문장 시나리오 명 |
| 검증 목적 | 무엇을 보장하기 위함인가 (예: 정책 조건, 골든 패스, 회귀 방지) |
| 우선순위 | P0(머스트, 출시 차단) / P1(중요) / P2(보통) / P3(여유 시) |
| 분류 | Smoke / Sanity / Regression / Exploratory / Performance / Security |
| 예상 소요 | 수동 검증 기준 분 단위 |
| 근거 / 리스크 | 왜 이 우선순위인지, 깨지면 어떤 임팩트가 있는지 |

## 작성 원칙
- **P0 = 출시 차단 사유** — 깨지면 절대 출시 불가한 케이스만
- 우선순위 산정은 **리스크 × 사용 빈도** 매트릭스 기반
- 시나리오는 단순 TC 의 묶음이 아니라 **사용자 여정 단위**
- 우선순위 분포는 일반적으로 P0 10~20% / P1 30~40% / P2 30% / P3 10~20% 가 균형적
- 회귀 영역 (변경 이후에도 깨지면 안 되는 기존 기능)을 빠뜨리지 말 것

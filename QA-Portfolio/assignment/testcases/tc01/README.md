# 과제 T-01 · 이메일 회원가입 테스트 케이스 작성

| 항목 | 내용 |
|---|---|
| 카테고리 | T · 테스트 케이스 작성 |
| 난이도 | 입문 |
| 권장 소요 시간 | 40분 |
| 대상 회사 | 일반 |
| 학습 목표 | 동등 분할 / 경계값 기법으로 TC 도출 |
| 평가 포인트 | Positive / Negative / Boundary 균형 · 우선순위 합리성 · 기대 결과 명확성 |

## 기능 명세
- 이메일, 비밀번호, 이름으로 회원가입한다.
- 이메일은 형식이 올바른 경우에만 가입 가능.
- 비밀번호는 **8자 이상 20자 이하**, 영문 / 숫자 / 특수문자 중 **2종류 이상** 포함.
- 이미 가입된 이메일은 재가입 불가.
- 이름은 **2자 이상 10자 이하**.
- 가입 완료 시 입력한 이메일로 환영 메일이 발송된다.
- 가입이 완료되면 사용자는 자동으로 로그인된 상태가 된다.

## 작성 문제
위 기능에 대한 테스트 케이스를 표준 양식으로 작성하시오.

## 작성 조건
- **최소 15개 이상** 작성
- Positive / Negative / Boundary / Negative-Boundary 균형 있게
- 경계값 케이스는 명시적으로 표기 (예: pw 7자 / 8자 / 20자 / 21자, 이름 1자 / 2자 / 10자 / 11자)
- 외부 시스템(메일 발송) 관련 TC 별도 분류
- 우선순위는 P0~P3 으로 분류하고, 모든 P0 케이스가 자동화 1순위가 될 수 있도록 선정

# 크림 (KREAM) · QA 채용 조사

> **조사일**: 2026-05-24
> **출처**: 공개 채용 공고, 채용 후기 사이트, QA 채용 관련 블로그
> **주의**: 사전테스트 실제 문항은 외부 유출 금지 정책으로 공개 자료 없음. 본 문서의 과제 유형은 **일반적인 QA 채용 사전과제 패턴 기반 추정**이다.

## 1. 회사 / 도메인 개요
- **도메인**: 한정판 거래 플랫폼 (네이버 자회사)
- **서비스 채널**: App (iOS / Android) + Web
- **주요 비즈니스 흐름**: 즉시 구매 / 구매 입찰 / 판매 입찰 / 검수 / 배송
- **QA 영역**: 서비스 + 백오피스

## 2. 채용 전형 절차

```
서류 → 사전테스트 + 1차면접 → 2차면접 → 컬처핏 검사 → 3차면접 → 채용검진 → 입사
```

| 단계 | 비고 |
|---|---|
| 서류 | 이력서 + 자기소개서 |
| 사전테스트 + 1차면접 | **같은 날 진행** — 사전 별도 과제가 아니라 **시간 제한형 필기 + 실무 인터뷰** 형태로 추정 |
| 2차면접 | 실무 중심 심층 면접 |
| 컬처핏 | 조직 적합도 검사 |
| 3차면접 | 임원 / 리더 면접 |
| 채용검진 | 건강검진 |

> 개발 직군은 프로그래머스 기반 코딩테스트를 별도 진행. QA 직군은 코딩테스트 대신 사전테스트 + 면접 형식으로 보이지만 정확한 형식은 비공개.

## 3. 모집 조건

### 자격 요건 (필수)
- QA 엔지니어 또는 관련 분야 **경력 3년 이상** (경력직 위주, 신입 채용은 거의 없음)
- 자기주도적 QA 진행 경험
- 애자일 / 스크럼 환경에서의 테스트 전략 수립 및 실행 경험

### 주요 업무
- App / Web 서비스 품질 유지·관리
- 백오피스 품질 관리
- 개발 요구사항 검토
- QA 산출물 작성 및 관리
- 결함 관리
- 테스트 케이스 / 시나리오 작성

## 4. QA 채용 사전과제 일반 유형

> 출처: [관리자용 QA 채용 과제와 필기전형 예시 (brunch.co.kr/@swtestrecipe/43)](https://brunch.co.kr/@swtestrecipe/43)
> 크림 특정 자료는 아니지만, 한국 IT 기업의 QA 사전과제·필기시험 패턴을 잘 정리한 자료.

### A. 사전과제 (Assignment) — `TC 작성 + α` 구조

| 유형 | 출제 형식 | 검증 항목 |
|---|---|---|
| **기본형** | 로그인 / 결제 같은 주요 기능 선택 → 유저 시나리오 기반 TC 작성 | TC 설계 능력 |
| **고급형** | 상태전이·경계값 오류 많은 기능 → 테스트 설계기법 적용한 전략적 TC | 설계 기법 응용 |
| **실무형** | 실제 사례 제시 → 테스트 수행 계획 + Scope 지정 + TC 작성 | 실무 대응력 |

### B. 필기전형 — 이론 + 실무

#### B-1. QA 공통 전문 지식
- 소프트웨어 공학 기본 이론
  - 프로젝트 제약 3요소 = **품질 · 비용 · 일정**
  - SDLC 단계 순서: 계획 → 요구분석 → 설계 → 구현 → 테스트 → 유지보수
- 테스팅 일반 원리
  - 예) **살충제 패러독스(Pesticide Paradox)** 설명 + 해결 방안
    - 개념: 동일 TC 반복하면 새 결함 발견 못 함
    - 해결: ① TC 주기적 개선 ② 탐색적 테스팅 / Ad-hoc

#### B-2. 모바일앱 / 실무 능력
- **테스트 설계기법 적용**
  - 예) "장보기 기능, 최대 3개 목록, 각 최대 50개 품목 → 경계값 분석 100% 커버 최소 TC"
  - 정답: `0 / 1 / 50 / 51개 품목` × `0 / 1 / 3 / 4개 목록`
- **테스팅 전략 수립**
  - 비즈니스 문제 제시(예: 앱 이탈률 높음) → "어디에 초점을 맞춰 테스트?"
  - 정답 예: "앱 시작 시간"
- **테스트 커버리지**
  - 결정 / 조건 커버리지 개념 설명 + 코드 기반 TC 작성

| 결정 / 조건 커버리지 | — | ❌ 없음 | **코드 기반 화이트박스 문제 없음** |

## 6. 보강 제안 (우선순위 순)

### ★ 우선순위 1: K (Knowledge / 이론 시험) 카테고리 신설
- 살충제 패러독스, 7가지 테스팅 원칙, SDLC, V-model
- Severity vs Priority, Black/White/Grey Box
- 단답형 + 서술형 혼합. 시간 제한 30~40분
- **사유**: 이론은 단순 암기로 점수 확보 가능. 사전테스트 시간 제한 환경에서 변별력 큼.

### ★ 우선순위 2: 실무형 통합 과제 신설
- 크림 입찰 출시 시나리오에 **테스트 계획서 + Scope + 핵심 TC 도출**을 한 문제에 묶음
- R + P + T를 결합한 케이스
- **사유**: 크림 같은 회사에서 실무 능력 변별 포인트. 60~90분 시간 제한.

### ★ 우선순위 3: T-02 상태전이 다이어그램 기반 TC
- 크림 거래 상태 흐름(`입찰 등록 → 거래 체결 → 결제 대기 → 발송 대기 → 입고 대기 → 검수 중 → 합격/보류/불합격`)을 상태전이도로 그린 뒤 TC 도출
- 상태전이 커버리지(0-switch / 1-switch) 명시
- **사유**: 크림은 상태전이가 풍부한 도메인이라 출제 가능성 높음.

### 우선순위 4: T-03 결정 / 조건 커버리지 (화이트박스)
- 코드 스니펫 제공 → 결정 커버리지 100%, 조건 커버리지 100%, MC/DC 100% 각각 만족하는 TC 작성
- **사유**: 화이트박스 기법 검증용. 출제 빈도는 중간.

### 우선순위 5: 사전테스트 모의 세트
- "60분 내 다음 3문제 풀이" 형식의 묶음 문서
- 예: 이론 5문제(15분) + 실무형 통합 1문제(45분)
- **사유**: 실제 사전테스트 환경 시뮬레이션.

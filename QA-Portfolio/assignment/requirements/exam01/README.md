## 안내
다음 요구사항을 읽고, 아래 5가지 기준에 따라 요구사항을 분석하시오.

## 기능명
서비스 신청/결제/승인 플로우

## 요구사항
- 사용자는 App/Web에서 특정 서비스를 신청할 수 있다.
- 사용자는 서비스 목록에서 원하는 서비스를 선택하고, 신청 정보를 입력한 뒤 결제를 진행한다.
- 결제가 완료되면 신청 상태는 “신청 완료”가 된다.
- 신청 완료 후 운영자는 신청 내용을 검토한다.
- 검토 결과는 승인, 보류, 반려로 나뉜다.
- 승인된 신청은 “진행 예정” 상태로 변경된다.
- 보류된 신청은 사용자에게 보류 사유가 노출되며, 사용자는 24시간 이내에 추가 정보를 입력할 수 있다.
- 사용자가 24시간 이내에 추가 정보를 입력하면 운영자는 다시 검토할 수 있다.
- 사용자가 24시간 이내에 추가 정보를 입력하지 않으면 신청은 자동 반려된다.
- 반려된 신청은 결제 취소 또는 환불 안내가 노출된다.
- 사용자는 신청 상태가 “신청 완료”일 때만 신청 정보를 수정할 수 있다.
- 신청 상태가 “검토 중”, “승인”, “보류”, “반려”일 때는 신청 정보를 수정할 수 없다.
- 사용자는 신청 내역 화면에서 현재 신청 상태를 확인할 수 있다.
- App과 Web에서 동일한 신청 상태와 안내 문구가 노출되어야 한다.

## 작성 문제
아래 5가지 기준으로 요구사항을 분석하시오.
1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

---

- 사용자: 구매자 / 판매자
- 주요기능: 특정 서비스 신청, 결제, 추가 정보 입력, 결제 취소, 신청 상태 확인/수정
- 상태변화
  - 신청 완료(승인된 신청) / 보류된 신청 
  - 검토결과: 승인, 보류, 반려
  - 진행 예정
- 예외 상황
- 외부 시스템
  - PG사 결제

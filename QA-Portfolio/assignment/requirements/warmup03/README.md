# 과제 W-03 · 게시글 작성 / 수정 / 삭제

| 항목 | 내용 |
|---|---|
| 카테고리 | R · 요구사항 분석 |
| 난이도 | 입문 |
| 권장 소요 시간 | 15분 |
| 대상 회사 | 일반 |
| 학습 목표 | 권한 기반 분기 표현 (로그인 / 비로그인, 본인 / 타인) |
| 평가 포인트 | 권한 매트릭스 빠짐없음 · 권한 없을 때의 UX · 필수 필드(제목/본문) 검증 |

## 안내
다음 요구사항을 읽고, 아래 5가지 기준에 따라 요구사항을 분석하시오.

## 기능명
게시글 작성 / 수정 / 삭제

## 요구사항
- 로그인한 사용자는 게시글을 작성할 수 있다.
- 게시글에는 제목과 본문이 모두 입력되어야 등록할 수 있다.
- 게시글 작성자는 본인이 작성한 글만 수정하거나 삭제할 수 있다.
- 게시글이 삭제되면 목록에서 즉시 사라진다.
- 비로그인 사용자는 게시글 목록과 상세는 볼 수 있지만, 작성·수정·삭제는 할 수 없다.

## 작성 문제
아래 5가지 기준으로 요구사항을 분석하시오.
1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

## 작성 조건
- 각 항목은 1개 이상 작성할 것
- 상태 변화는 “A → B” 형태로 작성할 것
- 로그인 / 비로그인 사용자의 권한 차이를 구분할 것
- 본인 글 / 타인 글의 권한 차이를 구분할 것

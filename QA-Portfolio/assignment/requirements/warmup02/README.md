## 안내
다음 요구사항을 읽고, 아래 5가지 기준에 따라 요구사항을 분석하시오.

## 기능명
단일 상품 장바구니 / 결제

## 요구사항
- 사용자는 상품 상세 화면에서 상품을 장바구니에 담을 수 있다.
- 장바구니에 담긴 상품의 수량은 1~10개 사이에서 조정할 수 있다.
- 사용자는 장바구니에서 상품을 삭제할 수 있다.
- 장바구니가 비어 있으면 결제 버튼은 비활성화된다.
- 결제 버튼을 누르면 결제 화면으로 이동한다.
- 결제가 완료되면 장바구니는 비워진다.
- 결제가 실패하면 장바구니의 상품은 그대로 유지된다.

## 작성 문제
아래 5가지 기준으로 요구사항을 분석하시오.
1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

## 작성 조건
- 각 항목은 1개 이상 작성할 것
- 상태 변화는 “A → B” 형태로 작성할 것
- 1~10개 같은 수치 제한을 놓치지 말 것
- 결제 성공 / 실패 케이스를 구분할 것

## 안내
다음 요구사항을 읽고, 아래 5가지 기준에 따라 요구사항을 분석하시오.

## 기능명
간편결제와 포인트/캐시백 및 환불 정산 플로우

## 요구사항
- 사용자는 가맹점 App/Web에서 간편결제로 상품을 결제할 수 있다.
- 결제 수단은 충전금(머니), 체크카드, 연결된 신용카드 중 선택할 수 있다.
- 사용자는 결제 시 보유 포인트를 추가로 차감할 수 있으며, 최소 사용 단위는 1,000원이다.
- 포인트는 결제 금액의 최대 50%까지만 사용할 수 있다.
- 결제가 완료되면 가맹점 카테고리에 따라 0.5%~3% 캐시백이 즉시 적립된다.
- 적립된 캐시백은 적립일 기준 7일 뒤 사용 가능한 포인트로 전환된다.
- 사용자는 결제 완료 후 30분 이내에 한해 본인이 직접 결제를 취소할 수 있다.
- 30분 경과 후에는 가맹점을 통해서만 환불 요청이 가능하다.
- 환불이 승인되면 결제 금액은 결제 수단에 따라 즉시(충전금) 또는 영업일 3~5일(카드) 내로 환불된다.
- 환불 시 사용된 포인트는 동일 금액으로 사용자에게 복원된다.
- 환불 시 적립되었던 캐시백은 회수되며, 잔액이 부족한 경우 마이너스 포인트로 남는다.
- 부분 환불이 가능한 가맹점의 경우 사용된 포인트와 적립된 캐시백은 환불 비율에 비례해 정산된다.
- 결제 직후 가맹점 서버가 응답하지 않으면 결제 상태는 결제 대기로 유지되며, 10분 후 자동 취소되고 출금된 금액은 환불된다.
- 1회 결제 한도는 100만원, 1일 결제 한도는 300만원이다.
- App과 Web에서 동일한 결제/환불 상태와 안내 문구가 노출되어야 한다.

## 작성 문제
아래 5가지 기준으로 요구사항을 분석하시오.
1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

## 작성 조건
- 각 항목은 3개 이상 작성할 것
- 상태 변화는 “A → B” 형태로 작성할 것
- 50% / 0.5%~3% 같은 비율 조건을 놓치지 말 것
- 7일 / 30분 / 영업일 3~5일 같은 시간 조건을 놓치지 말 것
- 결제 수단(충전금 vs 카드)에 따른 환불 경로 차이를 구분할 것
- 부분 환불의 포인트/캐시백 정산 로직을 별도 항목으로 작성할 것
- 가맹점 서버 무응답 케이스를 외부 시스템 또는 정책 조건에 포함할 것

## 안내
다음 요구사항을 읽고, 아래 5가지 기준에 따라 요구사항을 분석하시오.

## 기능명
영상 업로드와 자동 자막 생성 / 편집 / 내보내기 플로우

## 요구사항
- 사용자는 데스크톱 App에서 로컬 영상 파일(mp4, mov, avi)을 프로젝트로 불러올 수 있다.
- 단일 영상 파일 크기는 4GB 이하여야 한다.
- 영상이 프로젝트에 추가되면 시스템은 자동으로 음성을 분석해 자막을 생성한다.
- 자막 언어는 한국어, 영어, 일본어 등 사전 지정된 목록 중에서 선택할 수 있다.
- 자동 자막은 무음 구간을 기준으로 클립 단위로 분할되어 타임라인에 표시된다.
- 사용자는 각 클립의 자막 텍스트, 시작/종료 시점, 폰트/크기/색상을 수정할 수 있다.
- 사용자는 무음 구간 길이 임계값(기본 0.5초)을 설정해 무음 구간을 자동으로 제거할 수 있다.
- 자막을 수정해도 원본 오디오는 변경되지 않으며, 영상 길이는 클립 삭제/병합에 따라서만 변한다.
- 사용자는 텍스트 → 음성(TTS) 기능으로 자막을 AI 보이스로 더빙할 수 있다.
- TTS를 적용한 클립에서는 원본 오디오의 음소거 또는 볼륨 조절이 가능하다.
- 사용자는 결과물을 mp4(1080p / 720p / 480p) 또는 자막 파일(SRT, TXT) 단독으로 내보낼 수 있다.
- 내보내기 중에는 진행률이 노출되며, 사용자가 중간에 취소하면 생성 중이던 부분 파일은 자동 삭제된다.
- 사용자가 변경 사항을 저장하지 않고 프로젝트를 종료하려는 경우 저장 여부를 묻는 경고가 노출되어야 한다.
- 음성 분석은 인터넷이 연결된 상태에서만 동작하며, 오프라인 상태에서 추가된 영상은 분석 대기 상태로 보관되었다가 온라인 복구 시 자동 재시도된다.
- TTS는 월별 글자수 한도를 가지며, 한도를 초과한 사용 요청은 거부되고 안내 문구가 노출된다.

1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

## 작성 조건
- 각 항목은 3개 이상 작성할 것
- 상태 변화는 “A → B” 형태로 작성할 것
- 4GB / 1080p / 0.5초 / SRT 같은 구체 단위·규격을 항목에 포함할 것
- 자동 자막 생성 / 자막 편집 / 내보내기 단계를 분리해서 작성할 것
- 오프라인 상태, TTS 한도 초과, 내보내기 취소 같은 예외 흐름은 별도 항목으로 작성할 것
- 데스크톱 App 환경 특성(로컬 파일 접근, 저장되지 않은 변경)을 항목에 반영할 것
- 요구사항에 없는 외부 서비스명(특정 STT 엔진, 특정 클라우드)을 과하게 상상하지 말 것

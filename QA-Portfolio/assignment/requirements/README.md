# R · Requirements Analysis · 요구사항 분석

주어진 요구사항을 다음 5가지 기준으로 분해하는 연습.
1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

## 문제 목록

### 입문 (warmup) — 5가지 분석 기준 감 잡기
| 코드 | 시나리오 | 시간 | 대상 |
|---|---|---|---|
| [W-01](warmup01/README.md) | 이메일 회원가입 | 10분 | 일반 |
| [W-02](warmup02/README.md) | 장바구니 / 결제 | 15분 | 일반 |
| [W-03](warmup03/README.md) | 게시글 작성·수정·삭제 | 15분 | 일반 |

### 실전 (exam) — 회사별 도메인 함정 찾기
| 코드 | 시나리오 | 시간 | 대상 |
|---|---|---|---|
| [E-01](exam01/README.md) | 서비스 신청 / 결제 / 승인 | 30분 | 일반 |
| [E-02](exam02/README.md) | 한정판 입찰 / 검수 / 배송 | 35분 | 크림 |
| [E-03](exam03/README.md) | 송금 / 한도 / 본인 인증 | 40분 | 토스 |
| [E-04](exam04/README.md) | 간편결제 / 캐시백 / 환불 | 40분 | 토스 |
| [E-05](exam05/README.md) | Vrew 영상 자동 자막 / 편집 | 35분 | 보이저엑스 |

## 권장 순서
`W-01 → W-02 → W-03 → E-01 → E-02 → E-03 → E-04 → E-05`

## 평가 기준 (요구사항 분석 카테고리)
| 기준 | 가중치 | 평가 내용 |
|---|---|---|
| 완전성 | 30% | 5가지 기준 모두 작성, 항목 누락 없음 |
| 정확성 | 30% | 요구사항을 임의로 해석·추가하지 않음 |
| 분류력 | 20% | 정책 / 예외 / 외부 시스템 구분 정확 |
| 구체성 | 20% | 수치·상태·역할 명시 |

## 자주 빠뜨리는 체크리스트
- [ ] 시간 조건(분/시간/일) 누락 없음
- [ ] 금액 / 수량 / 비율 임계값 누락 없음
- [ ] 자동 처리(자동 반려·자동 환불) 별도 항목
- [ ] App / Web, 사용자 / 비사용자 같은 채널·역할 분기
- [ ] 부분 실패(예: 출금 성공 / 입금 실패) 흐름

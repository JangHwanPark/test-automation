## 안내
다음 요구사항을 읽고, 아래 5가지 기준에 따라 요구사항을 분석하시오.

## 기능명
계좌 송금 및 한도 관리 플로우

## 요구사항
- 사용자는 App/Web에서 본인 명의 계좌에서 다른 계좌로 송금할 수 있다.
- 송금 시 받는 사람의 은행과 계좌번호를 입력하면 시스템이 예금주명을 조회해 표시한다.
- 송금 금액은 1원 이상 1억원 이하여야 한다.
- 사용자는 1일 이체 한도(기본 1,000만원)와 1회 이체 한도(기본 200만원)를 가진다.
- 사용자는 본인 인증을 거쳐 한도를 상향 조정할 수 있다.
- 1회 이체 금액이 100만원을 초과하면 비밀번호 또는 생체 인증이 추가로 요구된다.
- 1회 이체 금액이 1,000만원을 초과하면 영상 통화 본인 인증이 요구된다.
- 입력된 계좌가 유효하지 않거나 예금주 조회에 실패하면 송금 요청은 진행되지 않고 출금도 일어나지 않는다.
- 송금 요청이 정상 접수되면 출금 → 입금 순으로 진행되며, 두 단계 모두 성공해야 송금 완료 상태가 된다.
- 출금은 성공했지만 입금이 실패한 경우 30분 이내에 자동으로 출금 금액이 사용자 계좌로 환불된다.
- 사용자는 송금 완료 직후 5분 이내에 한해 수취 거절을 요청할 수 있다.
- 수취인이 동일 서비스 사용자라면 즉시 알림이 전송되며 송금 완료 상태가 된다.
- 수취인이 서비스 미사용자라면 송금 요청은 24시간 동안 유효한 수령 링크로 전달되고, 미수령 시 송금이 자동 취소되며 환불 처리된다.
- 사용자가 보이스피싱 의심 계좌로 송금 시도 시 경고 화면이 노출되며 1분 대기 후에만 송금 진행이 가능하다.
- App과 Web에서 동일한 송금 상태와 안내 문구가 노출되어야 한다.

## 작성 문제
아래 5가지 기준으로 요구사항을 분석하시오.
1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

## 작성 조건
- 각 항목은 3개 이상 작성할 것
- 상태 변화는 “A → B” 형태로 작성할 것
- 100만/1,000만/1억 같은 금액 임계값을 놓치지 말 것
- 5분/30분/24시간 같은 시간 조건을 놓치지 말 것
- 수취인이 서비스 사용자 / 미사용자인 경우를 구분할 것
- 보이스피싱 의심 계좌 차단 같은 정책 조건은 외부 시스템 또는 정책 조건에 포함할 것
